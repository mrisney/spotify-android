Caching Images in Android

Introduction

This is a simple  image caching Android application based on an extensible pattern of cache eviction, the caching library was  developed first in this project Caching Algorithim.

The sample android application allows the user to search Google Images for a search term. Using the results, and parsing them with JSoup, a collection of Key-Value pair made up of ByteBuffers are placed into the cache as
Key -  the link src of the Google Search term as a ByteBuffer
Value - the binary .png file, as a ByteBuffer

Once these are loaded into cache, if there is a limit on the number of items that are configured in the cache, they are evicted.

There are 4 major eviction policies :
  • LRU - Least Recently Used
  • LFU - Least Frequently Used
  • FIFO - First In, First Out
  • SIze - the largest image, in bytes is evicted.
For the most part these eviction policies are built on Comparators in Java.
Each eviction policy, is built on a class, and added to an Enum, and then adding it to the Android UI, as a preferences.xml and so forth.

You can browse the javadocs for more information.
